1.幫學校註冊組完成的專案

有幫學校註冊組製作大學學歷歷程之網頁，因為是教育部傳送json檔案給註冊組，但註冊組的staff不知該如何處理 ，有緣透過教授的臉書得知此消息，因此身任此任務。 製作的內容為教育部傳送學生的個人資訊 以及 高中所參加活動製作的檔案 以及 成績 要分門別類照各個科系排序且製作成網頁，供教授來對各學生進行評分(大學申請入學)

在完成這個系統 我所經歷的流程為如下

資料處理：

    接收教育部傳送的 JSON 檔案。
     解析 JSON 檔案中的資料，包括學生的個人資訊、活動參加記錄和成績等。

資料存儲：
    將解析後的資料存儲到適當的資料庫中，我們是上傳至nas(檔案儲存裝置)，以便日後快速檢索和查詢。

網頁介面：
    使用 HTML、CSS 和 JavaScript 建立一個直觀且易於使用的網頁介面。
     將學生的個人資訊、活動記錄和成績以清晰的方式呈現在網頁上，以便註冊組的工作人員能夠輕鬆地瀏覽和管理。

使用者認證：
    實現nas之連結來確保使用者身份驗證功能，以確保只有授權的使用者才能夠訪問和管理學生的資料。

2.搶票程式

由於考試要訂房，利用f12之console之運作原理，利用Python編程語言和網頁開發工具完成搶票訂房，並且在特定時間就進入網站自動輸入personal info.便可訂房，之後也有順利訂到房間

在完成這個系統 我所經歷的流程為如下


分析網站結構：

    使用網路爬蟲技術，觀察目標網站的HTML結構，確定所需元素的位置和類型。

模擬瀏覽器行為：

    使用Python的Selenium庫，模擬瀏覽器行為，包括訪問網站、點擊按鈕、填寫表單等。

網站控制台分析：

    分析網站在網頁控制台中的JavaScript代碼，找到相應的表單提交方式或AJAX請求。

撰寫Python腳本：

    根據分析結果，撰寫Python腳本，使用Selenium庫模擬用戶操作，自動填寫表單並提交。

定時任務：

    使用Python的時間計時模塊，設置特定時間點執行腳本，以確保在考試開始前自動訂房。

錯誤處理：

    處理可能出現的錯誤情況，例如網絡連接問題、網站結構更改等，以確保程式的穩定運行。

測試和優化：

    進行測試，確保正確性和穩定性，並根據實際效果進行優化和改進。
